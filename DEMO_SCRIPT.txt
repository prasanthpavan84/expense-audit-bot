[0:30 – 0:50]  ARCHITECTURE WALKTHROUGH
================================================================================

(Show the architecture diagram or the Mermaid chart in the README.)

"Here's how it works:

 1. User input hits the Security Checkpoint first — it redacts credit
    cards, emails, and SSNs, and blocks prompt-injection attempts.

 2. Clean input goes to the Audit Orchestrator, which delegates to two
    specialist sub-agents:
    - Receipt Extractor: pulls structured data from the receipt text
    - Policy Verifier: checks amounts against corporate limits and
      vendor restrictions using the MCP Server

 3. A routing node decides the outcome:
    - Under $200 and compliant → auto-approved
    - Over $200 → pauses for human review
    - Policy violation → auto-denied

 Let me show you all three paths live."
