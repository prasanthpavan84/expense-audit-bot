# Before vs After Evaluation Framework Comparison

This report details the comparative audit of the evaluation framework's capabilities before and after remediation.

| Evaluation Metric / Feature | Before Remediation | After Remediation | Net Improvement / Value |
|---|---|---|---|
| **Subsystem observabilities** | Single aggregated OCR/Policy metrics | Isolated metrics for all 8 subsystems (Extraction, Validation, Fraud, Policy, etc.) | High resolution component tracking |
| **Statistical rigor** | Basic averages only | Precision, Recall, F1, Balanced Accuracy, Confusion Matrix | Peer-reviewable mathematical validation |
| **Telemetry tracking** | Missing Git and model metadata | Traced Git commit hash & model version in all JSON registries and Markdown files | Complete execution accountability |
| **Evidence confidence** | Implicit target assertions | Z-score computed 95% Confidence Intervals for latency | Measured latency boundaries |
| **Documentation completeness** | Missing schemas and readiness definitions | Full dataset summaries, readiness grading weights, manifest reference schemas | Transparent Capstone scoring |
| **Failure tracking** | Empty lists with no categories | Categorized failures & actual case studies printed to report | Granular debugging support |
